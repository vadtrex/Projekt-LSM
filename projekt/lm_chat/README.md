# lm_chat

Wymagania ogólne:

1.	W projekcie należy wykorzystać jeden z mechanizmów zarządzania stanem:
a.	Bloc
b.	Riverpod
2.	Aplikacja musi być przygotowana dla różnych rozmiarów ekranów:
a.	Telefony
b.	Tablety
3.	Projekt musi wykorzystywać co najmniej 2 zewnętrzne technologie. Przykłady:
a.	Firebase Authentication
b.	Firebase Firestore
c.	Firebase Storage
d.	REST API
e.	Google Maps API
f.	lokalizacja GPS
g.	kamera
h.	powiadomienia push / lokalne
i.	SQLite / Hive / Drift
4.	Aplikacja powinna zawierać minimum 5 różnych ekranów funkcjonalnych
5.	Projekt powinien być wykonany w sposób uporządkowany, z wyraźnym podziałem warstw i dobrą organizacją kodu (np. clean architecture)
